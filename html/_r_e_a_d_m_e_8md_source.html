4 ## Description
5 
6 The Bareflank Hypervisor is an open source, lightweight hypervisor, lead by
7 Assured Information Security, Inc. designed specifically to enable rapid
8 prototyping of hypervisor technologies. To ease development, Bareflank
9 is written in C++, and includes support for exceptions and the C++ Standard
10 Template Library (STL) via libc++. With the C++ STL, users can leverage
11 shared pointers, complex data structures (e.g. hash tables, maps, lists, etc…),
12 and several other modern C++ features. Existing open source hypervisors that
13 are written in C are difficult to modify, and spend a considerable amount of
14 time re-writing similar functionality instead of focusing on what matters
15 most: hypervisor technologies. Furthermore, users can leverage inheritance to
16 extend every part of the hypervisor to provide additional functionality above
17 and beyond what is already provided.
18 
19 To this end, Bareflank's primary goal is to provide the scaffolding needed to
20 construct more complete/complicated hypervisors including:
21 - Type 1 Hypervisors (like Xen)
22 - Type 2 Hypervisors (like VirtualBox)
23 - Host-Only Hypervisors (commonly used by anti-virus and rootkits)
24 
25 Although Bareflank aims to support as many hypervisor configurations as
26 possible, Bareflank itself will remain simple, and minimalistic; only providing
27 the scaffolding needed by these configurations. The core business logic
28 will remain in the hypervisors that extend Bareflank, and not in Bareflank
29 itself.
30 
31 To support this design approach, the entire Bareflank project is licensed under
32 the GNU Lesser General Public License v2.1 (LGPL), specifically enabling users
33 of the project to both contribute back to the project, but also create
34 proprietary extensions if so desired.
35 
36 In addition to Bareflank’s lightweight, modular design, the entire hypervisor
37 has been written using test driven development. As such, all of Bareflank’s
38 code comes complete with a set of unit tests to validate that the provided
39 code works as expected.

50 ## Compilation Instructions
51 
52 Before you can compile, you must have both a native GCC compiler, as well as a
53 GCC cross-compiler. If you are running on one of the supported platforms,
54 setting up the cross compiler is as simple as:
55 
56 ```
57 ./tools/scripts/setup-<platform>.sh
58 ```
59 
60 The setup-<platform>.sh script not only creates the cross compiler, but
61 it also sets up the libc and libc++ environment, creating a sysroot that will
62 be used by the Bareflank Hypervisor. Once you have your cross compiler setup
63 based on the script, you should be able to run the following:
64 
65 ```
66 make
67 make unittest
68 ```
69 
70 To run the hypervisor, you need to first compile, and load one of the driver
71 entry points. Bareflank uses the driver entry point to gain kernel level
72 access to the system to load the hypervisor. On Linux, this is as simple as:
73 
74 ```
75 make linux_load
76 make load
77 make start
78 ```
79 
80 to reverse this:
81 
82 ```
83 make stop
84 make unload
85 make linux_unload
86 ```
87 
88 to get status information, use the following:
89 
90 ```
91 make status
92 make dump
93 ```
94 
95 For more detailed instructions please read the following (based on which OS your using):
96 
97 [Driver Entry Documentation](https://github.com/Bareflank/hypervisor/tree/master/bfdrivers/src/arch)
