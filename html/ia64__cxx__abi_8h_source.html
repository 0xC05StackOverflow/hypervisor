34 // The IA64 C++ ABI specification describes a set of functions that are used
35 // to perform stack unwinding for exception support. This implementation uses
36 // the following documentation:
37 //
38 // http://mentorembedded.github.io/cxx-abi/abi-eh.html#cxx-abi
39 //
40 // Also note that this information can also be found in the System V 64bit
41 // ABI specification:
42 //
43 // http://www.x86-64.org/documentation/abi.pdf
44 //

46 // -----------------------------------------------------------------------------
47 // 1.1 Exception Handler Framework
48 // -----------------------------------------------------------------------------
49 
50 // The exception framework can be broken up into three layers:
51 //
52 // - Layer 3: Defines the language specific keywords for exception support.
53 // In C++, this is the "throw" and "catch" keywords that the programmer
54 // is actually using.
55 //
56 // - Layer 2: The compiler, using the layer 3 keywords, will automatically
57 // generate calls to layer 2 functions. These functions are provided by the
58 // ABI (for GCC this is libsupc++, and for LLVM, this is libcxxabi). These
59 // functions include __cxa_throw, __cxa_catch_begin, etc... These ABIs also
60 // provide the "personality" function that is used to determine which catch
61 // blocks we should stop unwinding to.
62 //
63 // - Layer 3: This layer defines the actual unwinding code. Layer two is an ABI
64 // specific abstraction layer, but layer 3 is actually performing the
65 // unwinding. This layer is usually provided by the compiler, as it is
66 // architecture specific. The problem with most implementations is, they
67 // require user space code to work. This one does not. The code defined here
68 // belongs to this layer

197 // -----------------------------------------------------------------------------
198 // 1.3 Throwing an Exception
199 // -----------------------------------------------------------------------------
200 //
201 // Throwing an exception can be done with two different functions:
202 // - _Unwind_RaiseException
203 // - _Unwind_Resume
204 //
205 // These functions are basically identical. They both get the current register
206 // state, and then unwind the stack until the personality function says to stop.
207 // The difference is, _Unwind_RaiseException is executed in two phases, a
208 // search phase and a cleanup phase, while _Unwind_Resume only performs the
209 // cleanup phase.
210 //
211 // It should be noted that if code contains RAII and has to preform cleanup,
212 // it's likely the _Unwind_RaiseException will be told to jump into the code,
213 // as if the "catch" block was identified, even though it wasn't the compiler
214 // will throw again by calling _Unwind_Resume. This code really doesn't
215 // know the difference between a real "throw, rethrow, catch" and a "throw,
216 // cleanup, catch" as they look identical
217 //
218

269 // -----------------------------------------------------------------------------
270 // 1.6 Personality Routine
271 // -----------------------------------------------------------------------------
272 //
273 // The personality routine is called by this code, and it tells us when to stop
274 // looking for a catch block. It uses information in the LSDA to determine if
275 // a catch block is in the current CFA. Layer 3 has to search in two phases.
276 // The first phase tells the personality function just to tell us when to stop.
277 // The second phase does the actual cleanup and then stops. Note that if a
278 // custom personality function was created, you could reduce this to a single
279 // cleanup phase, as the initial search phase is a not needed in most cases.
280 //
281 // Also note that since this is only meant to be used by C++, we don't need
282 // the force unwind functionality
283 //
